# Interactive Gated Workflow Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make a manual run of a `ticket-to-code` agent select the first Open Jira ticket assigned to the token owner, follow `workflows/jira-ticket-to-mr.md` with real interactive ⛔ gates (pause → operator approves on the dashboard → resume), and open a draft MR.

**Architecture:** Approach A — session-resume state machine. The runner drives the agent via `claude -p` with a stable per-run `--session-id`; the agent pauses at a gate by emitting a `<gate>{…}</gate>` block; the run parks in `waiting_approval`; the operator responds via a new endpoint; the runner resumes the same session with `claude -p --resume`. The workflow markdown + a gate-protocol preamble are injected into the system prompt.

**Tech Stack:** Node 20, TypeScript 5.7, Fastify 5 + TypeBox, Drizzle ORM + better-sqlite3, Jest (server tests), Vitest (runner tests), React + React Query (client), Claude Code CLI v2.1.190.

## Global Constraints

- Server tests: **Jest**, files in `apps/server/test/**/*.test.ts`. Run from `apps/server` with `npx jest <pattern>`.
- Runner tests: **Vitest**, files in `packages/runner/test/*.test.ts`. Run from `packages/runner` with `npx vitest run <pattern>`.
- DB migrations: generated by `npx drizzle-kit generate` (config `apps/server/drizzle.config.ts`); never hand-edit `meta/_journal.json`. New columns must be **nullable** (no backfill).
- The hand-written in-memory schemas in `apps/server/test/runs.test.ts`, `db.test.ts`, and `migration.test.ts` are separate from `schema.ts` — when adding a column used by a test, add it to that test's `CREATE TABLE` too.
- Secrets come from env vars only (`JIRA_API_TOKEN`, `JIRA_BASE_URL`, `GITLAB_API_TOKEN`). Never read/write/print a token.
- Jira status name is literal `"Open"`; project defaults to `CORE` (override `JIRA_PROJECT_KEY`). JQL: `project = <key> AND assignee = currentUser() AND status = "Open" ORDER BY created ASC`.
- Commit after each task. Branch: `feature/manual-run-jira-ticket-selection` (already created).
- Preserve the CLI subscription-auth model: keep stripping `ANTHROPIC_API_KEY`/`ANTHROPIC_AUTH_TOKEN`; never pass `--no-session-persistence`.

```ts
import { readFileSync, existsSync } from 'fs';
import { join } from 'path';

function stripFrontmatter(md: string): string {
  const m = md.match(/^---\s*\n[\s\S]*?\n---\s*\n?/);
  return m ? md.slice(m[0].length) : md;
}

export class WorkflowLoader {
  constructor(private workflowsDir: string) {}

  load(name?: string | null): string {
    if (!name) return '';
    const path = join(this.workflowsDir, `${name}.md`);
    if (!existsSync(path)) {
      console.warn(`[runner] workflow not found, skipping: ${name}`);
      return '';
    }
    try {
      return stripFrontmatter(readFileSync(path, 'utf-8').replace(/\r\n/g, '\n')).trim();
    } catch {
      console.warn(`[runner] workflow unreadable, skipping: ${name}`);
      return '';
    }
  }
}
```

```ts
export interface GatePayload {
  id: string; summary: string; question: string;
  kind: 'approve_reject' | 'input' | 'choice'; options?: string[];
}

export function extractGate(text: string): { gate: GatePayload | null } {
  const m = text.match(/<gate>([\s\S]*?)<\/gate>/);
  if (!m) return { gate: null };
  let parsed: GatePayload;
  try { parsed = JSON.parse(m[1].trim()); }
  catch { throw new Error(`Agent emitted a malformed <gate> block: ${m[1].slice(0, 200)}`); }
  if (!parsed.id || !parsed.question || !parsed.kind) {
    throw new Error(`Gate block missing required fields: ${m[1].slice(0, 200)}`);
  }
  return { gate: parsed };
}
```
