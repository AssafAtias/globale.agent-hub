# Activity Archive Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let users soft-archive individual runs from the monitoring dashboard's Activity feed, hiding them from the feed / NowStrip / health tiles by default, with a "Show archived" toggle to reveal and unarchive them.

**Architecture:** Add an `archived` boolean column to the `runs` table. The server exposes `PATCH /api/runs/:id` to flip the flag; `GET /api/runs` keeps returning all runs (filtering stays client-side, matching the existing dashboard). The client excludes archived runs from active/health/feed derivations and adds a per-row archive/unarchive button plus a "Show archived" toggle.

**Tech Stack:** Fastify 5 + TypeBox + Drizzle ORM (better-sqlite3) on the server; React 18 + MUI + TanStack Query on the client; vitest (client) and jest (server) for tests.

## Global Constraints

- Soft archive only — no hard delete, no bulk actions, no "older-than" cleanup.
- Filtering/derivation stays client-side; `GET /api/runs` returns all runs including the `archived` field.
- Archived runs are excluded from `selectActiveRuns`, `computeAgentHealth`, and (by default) `filterFeed`.
- No runtime DB migrator exists — migrations are applied manually to dev `.db` files.
- Follow existing repo patterns (TypeBox schemas on routes, `Type.Any()` responses, react-query `useMutation` with `['runs']` invalidation).

```ts
    app.patch('/api/runs/:id', {
      schema: {
        params: Type.Object({ id: Type.String() }),
        body: Type.Object({ archived: Type.Boolean() }),
        response: { 200: Type.Any(), 404: Type.Any() },
      },
    }, async (req, reply) => {
      const updated = RunRepository.setArchived(req.params.id, req.body.archived);
      if (!updated) return reply.status(404).send({ error: 'Not found' });
      return updated;
    });
```

In `apps/client/src/lib/runStats.ts`:

Add `showArchived` to `FeedFilter`:
```ts
export interface FeedFilter {
  agentId?: string;
  status?: string;
  showArchived?: boolean;
```

In `selectActiveRuns`, exclude archived:
```ts
export function selectActiveRuns(runs: Run[]): Run[] {
  return runs.filter((r) => !r.archived && ACTIVE.has(r.status)).sort(byCreatedDesc);
}
```

In `computeAgentHealth`, exclude archived from each agent's runs:
```ts
export function computeAgentHealth(runs: Run[], agents: Agent[]): AgentHealth[] {
  return agents.map((agent) => {
    const own = runs.filter((r) => r.agentId === agent.id && !r.archived).sort(byCreatedDesc);
    const done = own.filter((r) => r.status === 'done').length;
    const failed = own.filter((r) => r.status === 'failed').length;
    const running = own.filter((r) => ACTIVE.has(r.status)).length;
    const finished = done + failed;
    return {
      agent,
      total: own.length,
      done,
      failed,
      running,
      successRate: finished === 0 ? null : done / finished,
      lastRunAt: own.length > 0 ? own[0].createdAt : null,
      lastStatus: own.length > 0 ? own[0].status : null,
    };
  });
}
```

In `filterFeed`, drop archived unless `showArchived`:
```ts
export function filterFeed(runs: Run[], filter: FeedFilter): Run[] {
  return runs
    .filter((r) => (filter.showArchived ? true : !r.archived))
    .filter((r) => (filter.agentId ? r.agentId === filter.agentId : true))
    .filter((r) => (filter.status ? r.status === filter.status : true))
    .sort(byCreatedDesc);
}
```
